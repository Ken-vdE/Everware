# SEO + AI Indexation Optimization — Design

Date: 2026-07-14
Scope: existing one-page site only (NL `/` + generated EN `/en/`). No new content pages.

## Goal

Maximize discoverability and correct representation of everware.nl in:

1. Classic search engines (Google, Bing).
2. AI answer engines (Google AI Mode/Overviews, ChatGPT search, Perplexity,
   Claude, Copilot) — both their crawlers and their citation/preview surfaces.

## Constraints

- `index.html` (Dutch) is the source of truth; `en/index.html` is generated by
  `scripts/build-en.js` from `index.html` + the `copy` table in `assets/main.js`.
  All head/content changes must survive and be correctly translated by that pipeline.
- Phone `+31 6 12345678` is confirmed real — include in structured data.
- No street address on the site — do NOT add one to structured data.
- No social profiles exist — no `sameAs`.
- No reviews/ratings in schema (would be fabricated).
- Static hosting; no server-side code.

## Deliverables

### 1. JSON-LD structured data (`<script type="application/ld+json">` in head)

One `@graph` per language containing:

- **ProfessionalService** (`@id: https://everware.nl/#org`)
  - name Everware, url, email hallo@everware.nl, telephone +31612345678
  - `identifier`: PropertyValue { propertyID: "KvK", value: "89042786" }
  - `areaServed`: Netherlands; `knowsLanguage`: nl, en
  - `knowsAbout`: custom software, AI solutions, autonomous agents, process
    automation, LLM applications, RAG
  - `hasOfferCatalog`: 6 services matching the visible service cards
    (Procesoptimalisatie, Ontwerp & architectuur, Maatwerk software,
    AI oplossingen, Autonome agents, Advies) with descriptions.
  - `slogan` / `description` from existing meta copy.
- **WebSite** (`@id: …#website`) — name, url, publisher → #org, `inLanguage`.
- **WebPage** (`@id` per URL) — isPartOf #website, about #org, `inLanguage`.
- **FAQPage** — the 6 visible FAQ Q&As, verbatim from page copy.

Language handling: the Dutch block is baked into `index.html`. `build-en.js`
gains a `jsonld(lang)` step: it builds the English block from `copy.en` and
replaces the entire `<script type="application/ld+json">…</script>` block in
the generated page. FAQ/service texts are sourced from `copy` keys so EN never
drifts from visible EN text. A README rule covers keeping the NL block in sync
when FAQ/service copy changes (same file, same edit).

### 2. robots.txt

Keep `User-agent: * / Allow: /`. Add explicit allow stanzas (intent
documentation) for: GPTBot, OAI-SearchBot, ChatGPT-User, ClaudeBot,
Claude-SearchBot, PerplexityBot, Google-Extended, Applebot-Extended, CCBot,
meta-externalagent. Keep sitemap line. Add comment pointing to /llms.txt.

### 3. llms.txt (site root)

Markdown per the llmstxt.org convention: H1 "Everware", blockquote one-line
summary, sections for what the company does, services (6), how it works,
contact facts (email, phone, KvK, area served), and links to `/` (Dutch) and
`/en/` (English). Written in English (primary LLM consumption language) with a
note that the canonical site is Dutch.

### 4. Head metadata (both languages via build pipeline)

- `og:image` → `assets/og-image.png`, 1200×630, language-neutral: dark #050505
  background, dot grid, "Everware" wordmark, blue #3B82F6 accent, tagline
  "Software · AI · Agents". Rendered from a throwaway HTML file via headless
  Chrome. Plus `og:image:width`, `og:image:height`, `og:image:alt`,
  `og:site_name`.
- Twitter card: `twitter:card=summary_large_image`, `twitter:title`,
  `twitter:description`, `twitter:image` (per-language title/description via
  build-en.js, same as existing og handling).
- `<meta name="theme-color" content="#050505">`
- `<meta name="robots" content="index, follow, max-snippet:-1,
  max-image-preview:large, max-video-preview:-1">`

### 5. Semantic landmarks

- Wrap header nav links in `<nav aria-label="Hoofdnavigatie">`; `build-en.js`
  gets one targeted replace → `aria-label="Main navigation"`.
- Wrap the seven `<section>` elements in `<main>`.
- Wrap the footer block (bottom of contact section) in `<footer>`.
- Precondition: audit `assets/main.js` selectors so wrapping breaks nothing.

### 6. sitemap.xml

Add `<lastmod>2026-07-14</lastmod>` to both URLs. README note: bump on content
change.

### 7. Documentation (README)

- JSON-LD: where it lives, how EN block is generated, keep NL block in sync
  with visible copy.
- og-image: how to regenerate.
- llms.txt: what it is, update alongside content changes.
- sitemap lastmod rule.

## Explicitly out of scope

- New pages (services/blog).
- Street address / LocalBusiness geo data.
- sameAs social links, logo file beyond og-image, review/rating markup.
- Contact form backend.

## Verification

1. `node scripts/build-en.js` exits clean; `en/index.html` contains English
   JSON-LD with all keys translated.
2. JSON-LD blocks parse as JSON; structure spot-checked against schema.org
   types (ProfessionalService, WebSite, WebPage, FAQPage).
3. FAQ JSON-LD strings byte-match the visible FAQ copy per language.
4. Local serve + headless-Chrome screenshot of `/` and `/en/` — visual parity
   with pre-change screenshots (landmark wrapping broke no layout).
5. og-image.png exists at 1200×630 and head tags reference it on both pages.
6. robots.txt + llms.txt fetchable at root; sitemap validates as XML.
